1 IN-PERSON · 90 MIN Spawn Point
  • You are already a maker.
  • Verbs over tools — capabilities, not machines.
  • The make–notice–iterate loop.
  • The 8 capabilities of the Creator Commons: CUTPRINTSTITCHPRESSCASTPROTECTCAPTUREDESIGN
  • "Worked / broke / next" reflection format.
  • Pick one real, small annoyance from the past 7 days.
  • Translate it into a physical object you could prototype today.
  • Choose 2–3 verbs that combine into a 14-week project shape.
  • Naming a real user + real problem (not "students" / "an app").
  • Committing a shape under time pressure.
  • Honest reflection on a rough first build.
  • Capability Map 30 PTS
  • Cardboard Prototype 40 PTS
  • Project Seed 30 PTS
 Pre-college making. Roll-call wall surfaces childhood making (slime, doll clothes, cardboard forts, kitchen chemistry, stapled books) as proof the loop is already familiar. CS work is materially produced. Treating software as bodiless makes students brittle. Rooting them in physical making in Week 1 sets up iteration tolerance — the single biggest predictor of finishing.
2 ASYNC · 60 MIN Recon
  • You can't make more time — only see it and spend it.
  • A loadout is a system, not a list.
  • The 168-hour week.
  • "Compute time" vs. "wall time."
  • Fixed vs. discretionary blocks.
  • Tooling baseline: terminal, editor, browser profile, calendar.
  • Audit your real 168 hrs (not aspirational).
  • Find the 6–10 hrs/wk you'll spend on this project.
  • Install + verify the dev environment cold.
  • Fill a 168-grid honestly.
  • Run a "first command" in the terminal and screenshot it.
  • Post one intro in the cohort channel.
  • 168-hr Time Audit 20 PTS
  • Loadout Check 20 PTS
  • Cohort Intro Post 10 PTS
 ← The Project Seed's "v0.1 by Week 4" promise needs a real time budget. This week locates the hours where that build will actually happen. Setup hides a hidden curriculum. Students who never get unblocked here drop the major in semester 2. Making the loadout an explicit assignment levels the playing field for first-gen students.
3 LAB · 90 MIN Setup Quest
  • Version control is iteration, made durable.
  • Public work + private drafts.
  • init / add / commit / push, branches, PRs.
  • READMEs as designed artifacts.
  • The repo as portfolio surface.
  • Initialize a repo for your seed project.
  • Write a README that names the user, problem, and capabilities (lifted from your Project Seed).
  • Commit your cardboard-prototype photos as the first artifact.
  • One green-checkmark commit, on the right branch.
  • A README a stranger can parse in 30 seconds.
  • Photos checked in, not lost in camera roll.
  • Project Repo + README 40 PTS
  • First commit (cardboard photos) 10 PTS
 ← Pulls directly from the Project Seed (WHO/WHAT/which verbs) and Cardboard Prototype photos. The seed text becomes the README. Week 1 isn't archived — it ships into git. Iteration the field actually runs on. The make–notice loop from Week 1 maps cleanly onto commit–PR–review. Students who internalize this in Week 3 stop fearing rewrites for 4 years.
4 STUDIO · 90 MIN Project Seed v0.1
  • Ugly + early beats polished + late.
  • A v0.1 you can hand a user is the unit of progress.
  • Scope-cutting moves: smaller user, smaller problem, fewer verbs.
  • "Demo-able in 30 seconds" as the v0.1 bar.
  • Peer critique protocol: worked / broke / next.
  • Build a v0.1 using at least one of the verbs you marked "must" in the Capability Map.
  • Hand it to one classmate. Listen. Don't defend.
  • Write what you'd cut, keep, change.
  • A real object (or running prototype) a user can touch.
  • A 30-second demo, unrehearsed.
  • A scoped "next" that isn't bigger than v0.1 was.
  • v0.1 build + demo 50 PTS
  • Peer critique log 20 PTS
  • Revised seed (v0.2) 10 PTS
 ← Engages all three Week-1 artifacts at once: Capability Map chooses the verb, Cardboard Prototype set the iteration appetite, Project Seed defines the WHO and the v0.1 bar. Plus Weeks 2–3: hours are budgeted, repo exists. Most freshmen never ship in their first year of CS. Closing the spawn-to-v0.1 loop in 4 weeks is the entire purpose of this unit — it converts "I want to build things" into "I have built a thing, here's the next one."

SOURCE OF TRUTH

The Week-1 lesson page (spawn-point.html), facilitator guide, slide deck, and TILT rubric are all wired to the artifact names in column 6. Renaming an artifact means updating all four — start with the rubric.

WHEN THE PACING SLIPS

If Week 2's loadout breaks for >25% of the cohort, hold Week 3 a session and sub in a second Recon studio. Setup Quest assumes a working terminal — there's no clean way to teach git on top of broken Python.

WHAT EACH ROW IS NOT

This guide does not enumerate readings, slack-channel etiquette, or grading windows. Those live in the syllabus. The pacing guide answers a single question: does each week earn its place by engaging the previous?
