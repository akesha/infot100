DELIVERABLE 1 · WORKING STACK

Working stack.

50 PTS50
PURPOSE

Why this exists

CS curricula assume the dev environment Just Works by Week 1. It doesn't. By W4 you'll be expected to commit code to GitHub and reproduce a bug — neither happens without a verified install.
TASK

What to do

In class: complete three install rounds — terminal/editor, runtime/git, Obsidian — each with a verification command run in front of a TA. Push one empty repo to GitHub. Get TA initials in your Start Log for each round.
CRITERIA

How you're scored

Four criteria, each scored Exemplary / Proficient / Developing / Not yet. Total 50 pts. "Not yet" is fine for now — book office hours and you can re-verify.

DELIVERABLE 2 · EXTERNALIZED v0

Externalized v0.

50 PTS50
PURPOSE

Why this exists

Big Idea #4 — knowledge worth keeping must be externalized. Installing Obsidian and bookmarking the library doesn't externalize anything. Using them once does. v0 is the minimum artifact that proves the surface is real, not theoretical.
TASK

What to do

In class: write one note in Weeks/ and capture one credible source in Sources/. The source must come from an IU library database — not Google Scholar, not Wikipedia. Title + link + a one-sentence "why I trust this."
CRITERIA

How you're scored

Four criteria, each scored Exemplary / Proficient / Developing / Not yet. Total 50 pts. The hardest one to fake is the source rationale — that's the criterion that separates "I did it" from "I understood why."
